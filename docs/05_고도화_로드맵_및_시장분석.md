## Part 2. 시장 경쟁력 분석

---

### 0. 바이브코딩 시대가 바꾸는 게임의 규칙

> 2026년, AI Agent가 코드를 짜는 **바이브코딩(Vibe Coding)** 시대.  
> 개발 속도가 10~50배 빨라지면서 소프트웨어 시장의 경쟁 구도가 근본적으로 변했다.

| 구분 | 과거 (2020~2024) | 현재 (2025~2026) |
|------|-----------------|-----------------|
| **개발 속도** | 기능 1개 = 1~2주 | 기능 1개 = 0.5~1일 |
| **MVP 완성** | 3~6개월 | 1~2주 |
| **경쟁 장벽** | "먼저 만든" 기능 수 | **"얼마나 깊이" AI 통합했는가** |
| **인력 구조** | 팀 10~50명 | AI Agent + 소수 정예 1~5명 |
| **승부처** | 기능 완성도 | **AI 경험 깊이 + 실행 속도** |

**핵심 인사이트:**
- 기능은 누구나 빠르게 따라 만든다 → 기능 수로 차별화 불가
- **AI가 얼마나 깊이 업무를 실행하느냐**가 유일한 차별화
- BAIKAL은 이 전장에서 **AI-Native** 포지션을 선점한 상태

### 4. 핵심 경쟁 우위 (Competitive Moat)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   기존 그룹웨어 + AI (Post-hoc)     vs     BAIKAL (AI-Native)  │
│                                                             │
│   ┌──────────────────────┐         ┌──────────────────────┐│
│   │  기존 UI 그대로       │         │  AI가 UI를 대체      ││
│   │  + AI 챗봇 사이드 추가 │         │  대화 = 실행         ││
│   │  = "기능 하나 더"     │         │  = "패러다임 전환"    ││
│   └──────────────────────┘         └──────────────────────┘│
│                                                             │
│   😕 "AI 버튼 어디 있어?"           😊 "그냥 말하면 됨"      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**BAIKAL의 차별점은 "AI를 붙인 그룹웨어"가 아니라 "AI로 만든 그룹웨어"라는 점입니다.**

| 구분 | 경쟁사 (AI 추가 방식) | BAIKAL (AI 네이티브) |
|------|---------------------|---------------------|
| AI 역할 | 보조 도구 (검색, 요약) | **주 인터페이스** |
| 작업 방식 | 메뉴 + 폼 (AI는 옵션) | **대화 한 줄로 실행** |
| DB 연동 | AI가 텍스트만 생성 | **AI가 직접 DB CRUD** |
| 학습 비용 | 기존 UI + AI 별도 학습 | **말하면 됨 (교육 불요)** |
| 결과 | 2가지 경로 (UI / AI) 혼재 | **하나의 통합 경험** |
